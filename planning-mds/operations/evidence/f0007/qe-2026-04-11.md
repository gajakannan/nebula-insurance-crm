## Validation Coverage Summary

- **6-state workflow model** (Identified → Outreach → InReview → Quoted → Completed/Lost): All valid transitions and invalid pairs tested.
- **Role-gated transitions**: DistributionUser/Manager for intake stages, Underwriter for review stages, Admin for all. Denied transitions return `policy_denied`.
- **Conditional transition fields**: Lost requires `reasonCode` (6 valid codes), Other requires `reasonDetail`; Completed requires `boundPolicyId` or `renewalSubmissionId`. Service enforces `missing_transition_prerequisite`.
- **Optimistic concurrency**: `If-Match` header required for transitions and assignments; missing/stale returns 412.
- **One-active-per-policy**: Filtered unique index + service check prevents duplicate active renewals.
- **Per-LOB SLA thresholds**: WorkersCompensation=120d, Cyber=60d, default=90d seeded and used for urgency computation and TargetOutreachDate.
- **Stage-ownership validation**: `CanOwnRenewalStage` blocks non-Distribution users from owning Identified/Outreach and non-Underwriters from owning InReview/Quoted.

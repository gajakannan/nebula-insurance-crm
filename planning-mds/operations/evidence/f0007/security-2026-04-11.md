## Resource-Level Scoping (Service + Repository)

| Role | Scoping Rule | Implementation |
|------|--------------|----------------|
| Admin | Unscoped | `GetScopedQuery` returns all renewals; `CanReadRenewal` returns true |

| Underwriter | Assigned only | Same as DistributionUser |

**No data leakage paths identified.** List queries and detail reads both enforce scoping before returning data.

## Transition Authorization

Role comparison is case-insensitive via `StringComparer.OrdinalIgnoreCase`. Test coverage: `WorkflowStateMachineTests.ValidateRenewalTransition_ForDistributionUserIntoUnderwriterStage_ReturnsPolicyDenied`.

## Assignment Authorization

- Casbin: only `DistributionManager` and `Admin` have `renewal:assign` action.
- Service: `CanOwnRenewalStage` validates that the target assignee has the appropriate role for the current renewal stage (Distribution for intake, Underwriter for review).
- Terminal state renewals cannot be reassigned (`WorkflowStateMachine.IsTerminalState` check).
- No self-elevation path: non-managers cannot assign to others via the create flow (service check `CanCreateRenewal`).

**No SQL injection vectors:** All queries use EF Core LINQ parameterization. `AccountNameSearch` uses `EF.Functions.ILike` which parameterizes the search term. No raw SQL in service/repository code (only in migrations for data backfill and index creation).

## Concurrency Control

- Optimistic concurrency via PostgreSQL `xmin` column mapped to `RowVersion`.
- `If-Match` header required for transition and assignment endpoints.
- Missing/stale version returns 412 Precondition Failed.
- `DbUpdateConcurrencyException` caught in service for concurrent write detection.

## Audit Trail Integrity

- All mutations (create, transition, assign) emit `ActivityTimelineEvent` records with structured JSON payloads.
- Timeline events include `ActorUserId` from the authenticated principal — no actor spoofing possible.
- `WorkflowTransition` records are append-only (no update/delete interface exposed).
- Timeline inherits `renewal:read` scope — no unauthorized timeline access.

## Policy Stub Security

- `HasQueryFilter(e => !e.IsDeleted)` applied.
- FK relationships use `DeleteBehavior.Restrict` — no cascade deletion.
- PolicyNumber uniqueness enforced via unique index.

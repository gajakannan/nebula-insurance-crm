## 5) Blocking findings and remediation

1. API proxy target is hardcoded to localhost, breaking containerized E2E.
   - Evidence: repeated `ECONNREFUSED 127.0.0.1:5113` during Playwright container run.
   - File: `experience/vite.config.ts:8`
   - Required fix: make proxy target configurable (`VITE_API_PROXY_TARGET`) with CI/container-safe default, and align visual test runtime so frontend proxy can reach API.

2. Local OIDC dev flow is not reliably testable in current configuration.
   - Evidence:
     - Browser CORS block for discovery at `localhost:4173` origin.
     - Credentialed submit (`akadmin` / `devpassword`) at `localhost:5173` stayed on authentik flow and did not return to dashboard.
   - Files: `docker/authentik/blueprints/nebula-dev.yaml:92`, `docker/authentik/blueprints/nebula-dev.yaml:94`, `experience/playwright.config.ts:12`
   - Required fix: align local origins/redirect URIs and automate auth test setup (stable test account + deterministic login stage handling).

3. Initial dashboard mount triggers many breakdown requests, exceeding S0005 request-budget intent.
   - Evidence: Playwright container logs show many parallel `/dashboard/opportunities/.../breakdown` calls immediately on dashboard load.
   - File: `experience/src/features/opportunities/components/StageNodeStoryPanel.tsx:460`
   - Required fix: gate breakdown queries behind viewport/interaction/lazy toggles so initial mount stays within budget (target <= 6 parallel API requests).

4. Missing dedicated F0013 responsive/a11y Playwright coverage.
   - Evidence: visual suite currently contains `tests/visual/theme-smoke.spec.ts` and `tests/visual/f0012-dashboard-canvas.spec.ts`; no F0013-specific responsive gate exists.
   - File: `experience/tests/visual/theme-smoke.spec.ts:6`
   - Required fix: add F0013-specific Playwright specs for breakpoints, keyboard traversal, dialog behavior, reduced-motion, and chapter-mode transitions.
